# Move Workday to Backend Cron + Queue

## Context

Workday is currently fetched statelessly from the browser: the frontend POSTs to `https://{tenant}.wd{N}.myworkdayjobs.com/wday/cxs/{tenant}/{careerSite}/jobs` through the `api/workday.ts` Vercel CORS proxy, paginates `offset` against `total`, runs `workdayTransformer.ts` client-side, and never persists anything. This is the same browser-stateless shape Greenhouse had before `92bfdf6`, Ashby had before `1d0d95a` (PR #120), and that Gem (PR #121) / Lever (PR #122) are migrating right now.

Workday is **structurally distinct** from Gem/Lever/Ashby:

- **POST + JSON body** (not GET). Body is `{"appliedFacets": {}, "limit": 20, "offset": 0, "searchText": ""}`. Server caps `limit` at 20.
- **Pagination** uses `offset` against `total` from the first response. Cap at 100 iterations with a warning log.
- **Per-company config** ‚Äî needs `base_url`, `tenant_slug`, `career_site_slug` per company row. `companies.board_token` alone is insufficient. New JSONB `provider_config` column on `companies`.
- **Relative dates** ‚Äî `postedOn` is a string like `"Posted Today"` / `"Posted Yesterday"` / `"Posted 30+ Days Ago"`. The frontend uses `parseWorkdayDate` to produce ISO 8601 timestamps. Backend must mirror that exactly.

**Outcome:** Workday jobs land in `job_listings` with `source_id = 'workday_api'`. The 11 Workday entries in `companies.ts` flip to `backend-scraper` carrying `sourceAts: 'workday'`. The Why page renders Workday in its own column. `api/workday.ts`, `workdayClient.ts`, `workdayTransformer.ts`, `workdayDateParser.ts`, their tests, the `/api/workday/:path(.*)` vercel.json rewrite, and all `'workday'` / `WorkdayConfig` references in the frontend client-selection layer are removed.

## Repo Constraints (must follow)

- **Alembic autogenerate only.** Edit `src/backend/api/db_models.py` ‚Üí `alembic revision --autogenerate` for the schema change. Data migration body (INSERTs) is the documented exception. Memory: `feedback_use_alembic_migrations.md`.
- **No full-table rewrites.** The schema migration adds one column with a server default ‚Äî Postgres handles this as a metadata-only operation for new rows; existing rows pick up the default.
- **Bare table names.** `companies` and `job_listings` are env-agnostic.
- **Correctness over "don't crash"**: narrow exception handling in tasks (`httpx.HTTPError`, `ValueError`, `psycopg2.Error`); programmer errors propagate. Memory: `feedback_correctness_over_dont_crash.md`.
- **Async endpoints use `Depends(get_db)` and `asyncio.to_thread`** for any sync DB calls.
- **Admin gating**: every new trigger endpoint uses `Depends(require_admin)`.
- **`provider_config` column name is frozen** ‚Äî a parallel Eightfold PR will reuse it.

## Shared Contracts

These contracts are stable for the lifetime of this PR (and frozen for the sibling Eightfold PR):

- **`companies.provider_config`** ‚Äî `JSONB NOT NULL DEFAULT '{}'::jsonb`. Per-row shape is ATS-dependent. Workday rows MUST have keys: `base_url`, `tenant_slug`, `career_site_slug` (all strings, non-empty). Optional: `default_facets` (object mapping facet name to array of opaque ids).
- **`SourceId.WORKDAY = "workday_api"`** ‚Äî string constant in `scripts/shared/constants.py`. Imported by `workday_client.py`, `fetch_workday_company.py`, and tests.
- **Procrastinate queue `workday_fetch`** ‚Äî added to the worker's `queues=[...]` list in `main.py`.
- **Procrastinate task name `fetch_workday_company`** ‚Äî accepts kwargs `company_id`, `board_token`, `provider_config: dict`.
- **Procrastinate periodic id `workday_fan_out`** ‚Äî cron `*/30 * * * *`.
- **Queueing lock pattern `workday:{company_id}`** ‚Äî per per-company defer.
- **Admin trigger paths** ‚Äî `POST /api/jobs-qa/trigger-workday-fetch?company_id=...` and `POST /api/jobs-qa/trigger-workday-fan-out`, both `Depends(require_admin)`.
- **Frontend `sourceAts: 'workday'`** ‚Äî tagged on each migrated company config.
- **Why-page key `'workday'`** ‚Äî `ATS_DISPLAY_NAMES.workday = 'Workday'`, in `NON_CAPITALIZED_GROUPS`.

### Unit 3 ‚Äî `workday_client.py` (POST + paginate + transform)

**Owned files:**
- `src/backend/api/services/workday_client.py` (new)
- `src/backend/api/tests/test_workday_client.py` (new)

Tests cover: pagination loop terminates when total reached, terminates on empty page, terminates at cap (with ERROR logged), POST body shape is correct, default_facets passed through, transformer field mapping (every key documented above), date parser parity for all 4 input shapes plus None and garbage strings.

### Unit 4 ‚Äî `fetch_workday_company` Procrastinate task

**Owned files:**
- `src/backend/api/tasks/fetch_workday_company.py` (new)
- `src/backend/api/tests/test_fetch_workday_company.py` (new)

Mirrors `fetch_lever_company.py` structure: per-task psycopg2 connection via `asyncio.to_thread`, `httpx.AsyncClient` lifecycle, the same per-step auto-commit + retry idempotency order (upsert ‚Üí update_last_seen ‚Üí increment_consecutive_misses ‚Üí mark_jobs_closed), `scrape_runs` row in `finally`, narrow exception handling on `(httpx.HTTPError, ValueError, psycopg2.Error)`.

Function signature:
```python
@procrastinate_app.task(
    queue="workday_fetch",
    name="fetch_workday_company",
    retry=RetryStrategy(max_attempts=5, exponential_wait=2),
)
async def fetch_workday_company(
    company_id: str,
    board_token: str,
    provider_config: dict,
) -> None:
```

The `provider_config` arrives as a dict (Procrastinate serializes JSON). The task validates required keys (`base_url`, `tenant_slug`, `career_site_slug`) and raises `ValueError` if missing ‚Äî caught by the narrow `except` and recorded as a failed run (NOT a programmer error since this is data-driven).

Tests cover: happy path (1 page, 2 pages), safety guard trip, HTTP failure ‚Üí `error_count=1` + scrape_runs row, missing required `provider_config` keys ‚Üí `error_count=1`, connection-close error doesn't mask the underlying exception.

Commit: `Unit 4: add fetch_workday_company Procrastinate task`.
