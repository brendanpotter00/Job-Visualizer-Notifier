# Move Eightfold to Backend Cron + Queue

## Context

Eightfold is currently fetched statelessly from the browser: Netflix's `eightfoldClient.ts` walks `/api/apply/v2/jobs` via the `api/eightfold.ts` Vercel CORS proxy (which enforces an SSRF allowlist for `*.eightfold.ai` + vanity hosts like `explore.jobs.netflix.net`), runs `eightfoldTransformer.ts` client-side, and never persists anything. This mirrors the Greenhouse / Ashby / Workday situation that was already converted in PR #110 / #120 / #123 to a Procrastinate-backed backend pipeline.

This PR adapts the 10-unit shape from the Workday migration for Eightfold. Eightfold is structurally **smaller** than Workday (1 company today — Netflix — vs Workday's 11) but **structurally more interesting** in two ways:

1. **Sequential pagination with a hard 10-item server cap.** Empirically verified in `src/frontend/src/api/clients/eightfoldClient.ts` (`EIGHTFOLD_MAX_PAGE_SIZE = 10`). Requesting `num > 10` returns at most 10 positions. Netflix-sized tenants need ~60-100 requests, so the loop walks `start=0, 10, 20, …` until it hits a partial / empty page or `total`.

2. **SSRF allowlist.** `api/eightfold.ts` enforces that the `X-Eightfold-Tenant-Host` header must match `*.eightfold.ai` or one of a small vanity-host allowlist. This is the proxy's only security guarantee — once the proxy is deleted (Unit 7), that guarantee has to live somewhere. We port it to Python, in `eightfold_client.py`, so the per-company `tenant_host` from `provider_config` is rejected (with a `ValueError`) before any outbound HTTP if it falls outside the allowlist. This matches the Vercel proxy's failure mode (400 with "Invalid X-Eightfold-Tenant-Host value") and means a typo in a future seed migration can't be turned into an SSRF.

We re-use the **`provider_config` JSONB column on `companies`** introduced by the parallel Workday PR #123. That PR is currently open but unmerged. We assume it has not merged yet, so our Alembic migration adds the column AND seeds Netflix. If Workday merges first, the rebase is mechanical — drop the column-add half of our migration and keep the seed half. The column name is a frozen contract shared between the two PRs.

**Outcome:** Eightfold (Netflix) jobs land in `job_listings` with `source_id = 'eightfold_api'`. Netflix in `companies.ts` is flipped to `backend-scraper` with `sourceAts: 'eightfold'`. Why page renders Eightfold in its own column. `api/eightfold.ts`, `eightfoldClient.ts`, `eightfoldTransformer.ts`, their tests, the MSW handlers, and the SSRF-allowlist `vercel.json` rewrite are removed.

## Decisions Locked

| Decision | Choice |
|---|---|
| Scope | Eightfold only. |
| Companies count | **1** (Netflix). |
| Source ID | `"eightfold_api"` (matches Greenhouse + Ashby pattern). |
| Procrastinate queue | `"eightfold_fetch"`. |
| Cron frequency | `*/30 * * * *`. |
| Overlap policy | `queueing_lock=f"eightfold:{company_id}"`. |
| ID format | `str(position.get("id") or position.get("ats_job_id") or position.get("display_job_id"))`; drop if all three falsy. |
| `provider_config` shape | `{"tenant_host": str, "domain": str}` — both required, non-empty. |
| `posted_on` source | `_parse_eightfold_epoch(position.get("t_create"))` — UTC ISO; `None` on failure. |
| Schema migration | Autogenerated; combined with hand-written data half in same file (matches Workday's pattern). |
| Pagination cap | `MAX_PAGES = 100`; ERROR log + **partial-return** at cap (NOT raise — partial fetch is more useful than a zero-jobs run that would trip the safety guard). |
| SSRF allowlist | Ported to `eightfold_client.py`: regex `^(?:[a-z0-9-]+\.)*eightfold\.ai$` + `{"explore.jobs.netflix.net"}` set. Validated before any HTTP call. |
| `ATSGroupKey` | Already contains `'eightfold'`. No widening needed. Just update `ATS_DISPLAY_NAMES.eightfold = 'Eightfold'` (was lowercased) + add `'eightfold'` to `NON_CAPITALIZED_GROUPS`. |
| Workday rebase coordination | Plan assumes Workday NOT merged. Our migration adds `provider_config` column + seeds Netflix. If Workday merges first: drop the column-add half, keep the seed. Documented in DEPLOY.md. |

## Repo Constraints (must follow)

- Alembic autogenerate only (no hand-rolled schema files; data migrations are the exception).
- No `--no-verify` on commits.
- `Depends(get_db)` + `asyncio.to_thread` for sync DB in async endpoints.
- `Depends(require_admin)` on every new trigger endpoint.
- SSRF allowlist preserved in Python; reject any host outside `*.eightfold.ai` and the vanity host list **before** any HTTP call.
- Pagination cap at 100 pages with ERROR log + partial-return backstop.
- `provider_config` JSONB is a frozen contract shared with Workday PR #123.
- Narrow exception handling in tasks (`httpx.HTTPError`, `ValueError`, `psycopg2.Error` only); programmer errors propagate.

## Shared Contracts (frozen)

**HTTP — Eightfold fetch (paginated GET loop):**
```
GET https://{tenant_host}/api/apply/v2/jobs?domain={domain}&num=10&start={offset}
Headers: Accept: application/json
Timeout: 30s per request
Response: { "count": int, "positions": list[dict] }
```

**SSRF allowlist (Python):**
- Regex: `^(?:[a-z0-9-]+\.)*eightfold\.ai$` (case-insensitive)
- Vanity set: `{"explore.jobs.netflix.net"}`
- `_is_allowed_eightfold_host(host)` returns True iff host matches regex OR is in vanity set.

**`details` JSONB shape:**
```python
{
    "experience_level": position.get("experience_level"),
    "is_remote_eligible": bool(
        position.get("is_remote_eligible") or position.get("show_remote_eligibility")
    ),
    "department": position.get("department"),
    "team": position.get("team"),
    "canonical_position_url": position.get("canonicalPositionUrl"),
    "locations": position.get("locations"),
    "t_create_raw": position.get("t_create"),
}
```

**Procrastinate task signatures:**
```python
async def fetch_eightfold_company(
    company_id: str,
    board_token: str,
    provider_config: dict[str, str],
) -> None: ...

async def enqueue_eightfold_fan_out(timestamp: int) -> int: ...
```

**Queueing lock:** `queueing_lock=f"eightfold:{company_id}"`.

**`Company.sourceAts` (frontend):** widen `'ashby' | 'greenhouse'` → `'ashby' | 'greenhouse' | 'eightfold'`.
